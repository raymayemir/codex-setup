# Depth, Images, And Finishing

Use this reference when the structure is already sound and the interface needs atmosphere, emphasis, or polish.

## Depth Should Explain Elevation

- If you use shadows, define a small elevation system instead of inventing a new shadow for every component.
- Let the shadows imply a believable light source.
- Closer, more elevated elements should feel more prominent.
- Flat designs can still communicate depth through layered color, overlap, and contrast.

## Overlap With Intention

Overlapping panels, media, or cards can add hierarchy and momentum, but only when the overlap reinforces reading order or separation. Do not overlap elements just to make the composition busier.

## Images Need Structure Too

- Maintain reliable text contrast over images.
- Respect intended crop and display size.
- Be careful with user-uploaded imagery; it can break composition and contrast assumptions.
- If the UI depends heavily on images, design for ugly or low-quality assets as well as perfect ones.

## Finishing Moves That Usually Help

- improve default lists, dividers, and bullets instead of adding new decorative components
- use accent borders sparingly to call attention to important modules
- decorate selected backgrounds, not every background
- give empty states enough composition, guidance, and tone to feel intentional
- remove unnecessary borders when spacing or contrast can do the job

## Motion

- Motion should clarify entrance, relationship, hierarchy, or feedback.
- A few meaningful transitions are better than animation scattered everywhere.
- If motion does not improve understanding or delight, leave it out.

## Polish Comes Last

Do not use finishing touches to compensate for:
- weak grouping
- muddy hierarchy
- inconsistent type
- unclear action priority
- broken mobile layout

## Quick Checks

- Do shadows and overlap reinforce a coherent sense of depth?
- Are images resilient to real-world content?
- Do empty and sparse states still feel designed?
- Does polish increase clarity or mood without adding noise?
